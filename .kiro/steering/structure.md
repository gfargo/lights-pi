# Project Structure

## Root Files

- `lightsctl.sh` - Main control script (all Pi management commands)
- `Makefile` - Convenience shortcuts for lightsctl.sh commands
- `.env` - Local configuration (gitignored, copy from `.env.example`)
- `README.md` - Complete documentation

## Directories

### `/scripts`
Provisioning and configuration scripts executed on the Pi:
- `pi_lights_setup.sh` - Base installation (packages, QLC+, WiFi, systemd)
- `pi_harden.sh` - Security hardening (firewall, watchdog, udev)
- `pi_landing.sh` - Landing page setup (nginx)
- `configure_qlc_headless.sh` - Qt platform configuration for headless mode

### `/landing`
Optional landing page served at `http://lights.local`:
- `index.html` - Simple branded page with link to QLC+ web UI

### `/workspaces`
QLC+ workspace files (`.qxw` format):
- Store fixture configurations, scenes, and virtual console layouts
- Deploy to Pi with `./lightsctl.sh deploy-workspace <file.qxw>`

### `/backups`
Local backup storage (gitignored):
- Created by `./lightsctl.sh backup`
- Contains timestamped `.tar.gz` archives of Pi's QLC+ config

### `/certs`
TLS certificates for HTTPS (gitignored):
- Generated by `./lightsctl.sh gen-cert`
- Used by `./lightsctl.sh ssl-proxy` for stunnel configuration

## Conventions

### Script Execution Pattern
All Pi operations go through `lightsctl.sh`, which:
1. Loads `.env` configuration
2. Constructs SSH commands with proper credentials
3. Executes remote commands or transfers files
4. Provides consistent error handling and output

### Remote Script Execution
Provisioning scripts in `/scripts` are:
1. Transferred to Pi via SCP
2. Executed with sudo via SSH
3. Cleaned up after completion

### Service Management
QLC+ runs as `qlcplus-web.service` systemd unit:
- Starts on boot
- Runs in headless mode with `QT_QPA_PLATFORM=minimal`
- Logs to systemd journal
- Crash loop protection (max 5 restarts in 60s)

### Configuration Files on Pi
- `/etc/systemd/system/qlcplus-web.service` - QLC+ service definition
- `/etc/wpa_supplicant/wpa_supplicant.conf` - WiFi configuration
- `/etc/ufw/` - Firewall rules
- `/etc/udev/rules.d/99-dmx.rules` - ENTTEC USB symlink
- `/home/pi/.config/qlcplus/` - QLC+ user configuration
- `/home/pi/.qlcplus/` - QLC+ data directory
