# Project Structure

## Root Files

- `lightsctl.sh` - Main control script (all Pi management commands)
- `Makefile` - Convenience shortcuts for `lightsctl.sh` commands
- `.env` - Local configuration (gitignored, copy from `.env.example`)
- `README.md` - User-facing documentation

## Top-Level Directories

### `/scripts`

Provisioning, library helpers, and remote scripts.

- `scripts/provisioning/` - one-shot setup and hardening scripts run remotely
  on the Pi (`setup.sh`, `harden.sh`)
- `scripts/services/` - service installers run remotely (`landing.sh`,
  `control_server.sh`, `wifi-watchdog.sh`)
- `scripts/lib/` - shared bash libraries sourced by `lightsctl.sh` for
  organized command implementations
  - `system.sh`, `qlc.sh`, `wifi.sh`, `network.sh`, `backup.sh`, `tls.sh`
  - `ai_scene.sh` - AI scene-generation prompt construction and API calls
  - `ai_scene_mock.sh` - mock generator for offline testing
  - `scene_templates.sh` - pre-defined template generation
  - `fixture_groups.sh` - group CRUD + group-scoped scene/template generation
  - `workspace.sh` - inject/extract scenes from `.qxw` workspaces
  - `extract_fixtures.py` - Python helper that emits enriched fixture JSON for
    AI prompts using the `.qxf` parser
- `scripts/debug/` - one-off diagnostic scripts

### `/control-server`

Flask + SocketIO server running on the Pi at port `5000`.

- `app.py` - all HTTP routes plus the persistent QLC+ WebSocket plumbing
- `fixture_definitions.py` - parses QLC+ `.qxf` fixture files from
  `/usr/share/qlcplus/fixtures/` and `~/.qlcplus/fixtures/` into role-aware
  channel metadata. Used by both the control server and `extract_fixtures.py`.
- `templates/index.html` - the live control UI (AI chat, virtual console,
  fixture groups, channel sliders)
- `requirements.txt` - Python dependencies (Flask, Flask-CORS,
  Flask-SocketIO, websockets, requests)
- `README.md` - control server architecture notes

### `/landing`

Optional landing page served at `http://lights.local`:

- `index.html` - branded splash with link to the control server / QLC+ web UI

### `/scenes`

Reference scene XML samples (commit-tracked); `scenes/examples/` holds known-
good AI outputs used as regression / inspiration material.

### `/marketing-site`

Next.js 15 app for the public website (`https://riversway.studio` etc).
Editor-shown as `.www/` in some references — same directory.

### `/backups`

Local backup storage (gitignored). Created by `./lightsctl.sh backup`.

### `/certs`

TLS material (gitignored). Generated by `./lightsctl.sh gen-cert` or
`gen-cert-mkcert`; consumed by `setup-ssl` / `ssl-nginx` / `ssl-proxy`.

### `/docs`

Long-form architectural and feature docs.

## Conventions

### Script Execution Pattern

All Pi operations go through `lightsctl.sh`, which:

1. Loads `.env` configuration (with macOS-friendly `load_env_file` that does
   not collide with shell builtins like `HOSTNAME`)
2. Constructs SSH commands with proper credentials (`SSH_KEY` if set)
3. Sources the appropriate `scripts/lib/*.sh` library and calls into it
4. Provides consistent error handling and output

### Remote Script Execution

Provisioning scripts in `scripts/provisioning/` and `scripts/services/` are:

1. Transferred to Pi via SCP
2. Executed with sudo via SSH
3. Cleaned up after completion

### Service Management on the Pi

Two systemd services run on the Pi:

- `qlcplus-web.service` - QLC+ in headless mode, port 9999
  - `QT_QPA_PLATFORM=minimal` for true headless operation
  - Auto-loads workspace from `/home/<user>/.qlcplus/default.qxw`
  - Crash loop protection (max 5 restarts in 60s)
- `lighting-control.service` - the Flask control server, port 5000
  - Holds a single persistent WebSocket to QLC+ at `ws://localhost:9999/qlcplusWS`
  - Reads `.env` from `/home/<user>/.env` for AI provider config
  - Auto-restarts on failure

### Configuration Files on Pi

- `/etc/systemd/system/qlcplus-web.service` - QLC+ service definition
- `/etc/systemd/system/lighting-control.service` - control server service
- `/etc/NetworkManager/system-connections/` and `/etc/netplan/` - WiFi
  (NetworkManager is the active manager on Bookworm; `wpa_supplicant.conf` is
  vestigial and only relevant on older images)
- `/etc/ufw/` - firewall rules (allow 22, 80, 443, 5000, 9999)
- `/etc/udev/rules.d/99-dmx.rules` - ENTTEC USB symlink
- `/home/<user>/.qlcplus/default.qxw` - the live workspace
- `/home/<user>/.qlcplus/fixture_groups.json` - persisted fixture groups
- `/usr/share/qlcplus/fixtures/<Manufacturer>/<Model>.qxf` - fixture defs
- `~/.qlcplus/fixtures/` - user-installed fixture defs (override system)
